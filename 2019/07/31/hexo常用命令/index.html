hexo常用命令

Posted on 2019-07-31 Edited on 2020-05-27 In web
Symbols count in article: 1.4k Reading time ≈ 1 mins.

本文简介

博客框架hexo 常用到的一些命令

hexo简介

主页： https://hexo.io/zh-cn/
  Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

hexo常用命令

hexo version

显示 Hexo 版本

hexo new [layout] <title>

hexo new “我的第一篇文章”
hexo new post “我的第一篇文章”
hexo new draft “我的草稿”
hexo new page “tags”
hexo new page “tags”

hexo generate

生成静态文件,简化命令 hexo g
-d, –deploy 文件生成后立即部署网站
-w, –watch 监视文件变动

hexo publish [layout] <filename>

发布草稿

hexo server

启动hexo服务

hexo deploy

部署站点到git服务器上，命令简写： hexo d
-g, –generate 部署之前预先生成静态文件

hexo clean

清除缓存文件 (db.json) 和已生成的静态文件 (public)。

hexo –debug

在终端中显示调试信息并记录到 debug.log。当您碰到问题时，可以尝试用调试模式重新执行一次，并 提交调试信息到 GitHub。

hexo –draft

显示’source/_drafts’ 文件夹中的草稿文章。

hexo server –drafts

希望强行预览草稿，或者更改配置文件：
render_drafts: true

hexo publish [layout] <filename>

发布草稿，filename中不含’.md’

组合使用

$hexo clean && hexo g && hexo s
重新生成静态文件并启动服务

模版（Scaffold）

在scaffolds目录下新建模板photo.md,编辑photo.md，然后就可以根据模板新建文章了，生成的文件存放在 source/_posts中。
hexo new photo “My Gallery”

创建分类

$ hexo new page categories
修改配置文件：
$hexo_home/source/categories/index.md
添加：
type: “categories”

创建标签

$ hexo new page tags
修改配置文件：
$hexo_home/source/tags/index.md
添加一行：
type: “tags”

列出站点信息

type取值： page, post, route, tag, category
$ hexo list <Type>

安装其他插件

npm install hexo-deployer-git –save
npm install hexo-generator-index –save
npm install hexo-generator-archive –save
npm install hexo-generator-tag –save
npm install hexo-generator-sitemap –save
npm install hexo-baidu-url-submit –save
npm install hexo-generator-searchdb –save
npm install hexo-word-counter –save
